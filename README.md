# project-bootstrap

このフォルダは、**AI 駆動開発でプロジェクトを立ち上げるときに最初に入れる Skill** をまとめたものです。

## これは何か

この Skill は、AI にコードを書かせる前提で、

- どういう順番で開発を進めるか
- どのように設計するか
- どのようにディレクトリを切るか
- どのようにテストを組み込むか
- バグ修正をどう進めるか

を規定するための **開発運用ルール** です。

単なるコーディング規約ではなく、**設計 → 実装 → 検証 → 修正** まで含めた上位ルールです。

## なぜ README を置くのか

`SKILL.md` は基本的に **AI に読ませるためのファイル** です。  
そのため、命令的・簡潔・構造的に書かれており、人間が最初に読む説明書としては少し不親切です。

この `README.md` は人間向けに、

- この Skill の目的
- どういう場面で使うか
- 何を期待しているか
- どう運用すべきか

を分かりやすく説明するために置いています。

## いつ使うか

この Skill は、次のようなタイミングで使います。

- 新しいプロジェクトを開始するとき
- 既存プロジェクトに AI 駆動開発の運用ルールを導入したいとき
- AI と壁打ちしながら設計を進めたいとき
- 設計だけでなく、実装・テスト・修正まで一貫したルールを持たせたいとき

## 何を規定しているか

`SKILL.md` では主に以下を規定しています。

- 開発の基本原則  
  - KISS
  - Clean Architecture
  - SOLID
  - Fail Fast
  - Root Cause Fix
  - Test-Guided Development

- 開発フロー  
  - 目的整理
  - 要求整理
  - 設計
  - テスト観点整理
  - 小さく実装
  - 検証
  - 根本修正

- ディレクトリ構成方針  
  - レイヤー主軸
  - 必要に応じて bounded context / feature でサブ構造化

- AI への依頼ルール  
  - 小さい責務単位で依頼する
  - 推測実装を禁止する
  - テスト観点を先に定義する
  - 完了条件を明示する

- 実装前 / 実装後のチェック項目
- 完了条件

## この Skill の位置づけ

これは「人間が毎回全部読む長文ドキュメント」ではなく、  
**プロジェクトの最上位の運転ルールを AI と人間で共有するためのファイル** です。

役割としては次に近いです。

- 憲法
- 開発の operating rules
- AI に渡す前提条件
- プロジェクト起動時の共通ルール

## 運用方法

推奨運用は次の通りです。

1. プロジェクト開始時に `skills/project-bootstrap/` を配置する
2. AI との最初の壁打ち前に `SKILL.md` を前提として扱う
3. 要求整理・設計・実装・修正の各段階でこの Skill に従う
4. 細かい案件ごとの仕様は別の docs に追加する

## 推奨配置

```text
project-root/
  skills/
    project-bootstrap/
      README.md
      SKILL.md
```

## 補足

- `README.md` は人間向け説明
- `SKILL.md` は AI 向け運用ルール

という分担にしておくと分かりやすいです。

この分け方はかなり相性が良く、実運用でも自然です。
