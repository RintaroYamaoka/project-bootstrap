# /red — TDD Red phase entry

タスク: $ARGUMENTS

このコマンドは TDD の Red フェーズを開始する。

**`test-writer` サブエージェントを起動して**、上記タスクに対する failing テストを書いてもらう。

## サブエージェントに渡す情報

- タスク (上記)
- 関連するファイル / モジュール (現在の context から特定)
- テストフレームワーク / 実行コマンド (プロジェクトの CLAUDE.md または既存テストから判断)

## エージェント実行後の確認事項

- テストファイルが書かれた
- テストが期待通り fail する (別の理由で fail していない、また pass していない)
- 次の Green フェーズに必要な情報が揃っている (実装すべき関数・期待入出力・実行コマンド)

## 次のステップ

`/green <ターゲットテスト>` で Green フェーズに移行する。
